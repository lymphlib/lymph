\section secUGL1 Setup and Configuration

  A single simulation in lymph use the following MATLAB scripts:
  - \b SimulationSetup.m   : setup and configuration
  - \b RunSetup.m          : run setup and configuration
  - \b RunMainLaplacian.m  : run simulation 

  The different proposed testcases are the following MATLAB scripts:
  - \b DataTestLap.m               : data input file 
  - \b DataTestTanh.m              : data input file for adaptive simulation
  - \b DataConvTestLaph.m           : data input file for convergence test with respect to the mesh refinement
  - \b DataConvTestLapp.m           : data input file for convergence test with respect to the polynomial degree
  - \b DatapAdaptiveConvLap.m      : data input file for convergence test for adaptive simulation
  - \b DataComparisonpAdaptive.m   : data input file for comparison convergence test between adaptive and uniform simulation

   Other secondary scripts are:
  - \b RunhConvergenceLaplacian.m : verification test for h convergence (see \ref secTutLapConvh)
  - \b RunpConvergenceLaplacian.m : verification test for p convergence (see \ref secTutLapConvp)
  - \b RunNDoFConvergencep.m      : verification test for NDoF convergence for adaptive framework (see \ref pAdaptiveConv)


\subsection subUGL1 Simulation Setup

The users can configure or customize the code for their specific needs.
In particular, \b RunSetup.m defines the  simulation setup and contains the following fields:
    \verbatim
    - isPlotMesh    : 1/0 to plot the polygonal mesh
    - isSaveVTKMesh : 1/0 to save the polygonal mesh in the VTK format
    - isPlotSolution: 1/0 to visualize the numerical solution in Matlab
    - isSaveSolution: 1/0 to save the solution in the .mat format
    - OutFolder     : folder name for the output (e.g. Output)
    - isSaveCSV     : 1/0 to save the solution in the .csv format
    - isSaveVTK     : 1/0 to save the solution in the .vtk format
    - OutFolderVTK  : folder name for the VTK-output
    - isError       : 1/0 to compute the error (e.g. OutputVTK)
    - isParallel    : 1/0 for parallel implementation
    - numCores      : define number of cores for parallel implementation
    \endverbatim

The users can configure the initialization of the simulation through the 
\b SimulationSetup.m script. This file imports the required paths, initializes 
the simulation environment, loads the setup structure and 
manages the parallel computing settings. In particular, the script can 
automatically start or close a parallel pool depending on the user-defined 
options contained in the setup structure.
    \verbatim
%% Import lymph and add path related to this physics.
run("ImportLymphPaths.m")
addpath(genpath(MyPhysicsPath));
%% Print the header
Header;
%% Create the setup structure
run("RunSetup.m")
%% Initiate the parallel
Setup.p = gcp('nocreate');
if Setup.isParallel && isempty(Setup.p)
    parpool(Setup.numCores);
elseif not(Setup.isParallel) && ~isempty(Setup.p)
    delete(Setup.p)
end
ps = parallel.Settings;
ps.Pool.AutoCreate = false;
    \endverbatim
\subsection subUGL2 Input data files

To define the input data for the specific simulation you can use a MATLAB script (e.g. \b InputData/DataTestLap.m).
You can set up the number of the element mesh, the polynomial approximation degree, the penalty constant,
boundary conditions, forcing terms and analystical solution (if needed). 

 \verbatim
%% InputData/DataTestLap.m 
%% Poisson problem in [0,1]^2 with Dirichlet conditions
Data.name = 'TestLapDir';
Data.LabEl = {'L'};     % Element labels
Data.TagEl = { 1 };     % Element tags
Data.TagBc = {[2 3 4 5]};  % Boundary tags
Data.LabBc = {'DDDD'};  % (D)irichlet/(N)eumann/(A)bsorbing
%% Geometrical properties 
Data.domain       = [0 1 0 1]; % domain bounds for a new mesh
Data.N            = 1600;      % number of elements for a new mesh
Data.MeshFromFile = true;      % read mesh from file
...
%% Space discretization
Data.degree  = 4;        % Polynomial degree
Data.penalty_coeff = 10; % Penalty coefficient
...
%% Material properties 
Data.mu       = {@(x,y) 1.*x.^0.*y.^0};
% Forcing Term
Data.source   = {@(x,y) 10*(4*pi^2)*sin(2*pi*x).*cos(2*pi*y)};
% Boundary Conditions
Data.DirBC    = {@(x,y) 5*sin(2*pi*x).*cos(2*pi*y)};
% Gradient of the Exact Solution
Data.gradDirBC =  {@(x,y)  10*pi*cos(2*pi*x).*cos(2*pi*y); ...
                  @(x,y) -10*pi*sin(2*pi*x).*sin(2*pi*y)};
% Exact Solution (if any)
Data.u_ex     =  {@(x,y) 5*sin(2*pi*x).*cos(2*pi*y)};
% Gradient of the Exact Solution
Data.gradu_ex =  {@(x,y)  10*pi*cos(2*pi*x).*cos(2*pi*y); ...
                  @(x,y) -10*pi*sin(2*pi*x).*sin(2*pi*y)};
%% Adaptivity
Data.Adaptivity = false;                 % Flag of adaptivity                                             
...                  

\endverbatim

To define the input data for the \f$\ell\f$-adaptive simulation, you can use a MATLAB script 
(e.g. \b InputData/DataTestTanh.m). In this test case, the problem is solved on the unit square 
with Dirichlet boundary conditions, using a mesh loaded from file. The script specifies the mesh, 
the polynomial degree, the quadrature rule, visualization options, forcing term, boundary data, 
exact solution and the parameters controlling the adaptive procedure.

 \verbatim
%% InputData/DataTestTanh.m 
%% Adaptive Poisson problem in [0,1]^2 with Dirichlet conditions
Data.name = 'AdaptLap';
Data.LabEl = {'L'};     % Element labels
Data.TagEl = { 1 };     % Element tags
Data.TagBc = {[2 3 4 5]};  % Boundary tags
Data.LabBc = {'DDDD'};  % (D)irichlet/(N)eumann/(A)bsorbing
%% Geometrical properties 
Data.domain       = [0 1 0 1]; % domain bounds for a new mesh
Data.N            = 180;     % number of elements for a new mesh
Data.MeshFromFile = true;      % read mesh from file
Data.FolderName   = 'InputMesh';
Data.VTKMeshFileName = 'Mesh.vtk';
Data.meshfileseq  = 'Lap_180_el.mat'; %filename for mesh

%% Space discretization
Data.degree  = 6;        % Polynomial degree
Data.penalty_coeff = 10; % Penalty coefficient
...
%% Material properties 
Data.mu       = {@(x,y) 1.*x.^0.*y.^0};
% Forcing Term
Data.source   = {@(x,y) -80*(sech(20*(-0.8+x.^2+y.^2))).^2.*(-1+40*(x.^2+y.^2).*tanh(20*(-0.8+x.^2+y.^2)))};
% Boundary Conditions
Data.DirBC    = {@(x,y) tanh(-20*(x.^2+y.^2-0.8))};
% Gradient of the Exact Solution
Data.gradDirBC =  {@(x,y) -40*x.*sech(20.*(x.^2+y.^2-4/5)).^2; ...
                   @(x,y) -40*y.*sech(20.*(x.^2+y.^2-4/5)).^2};
% Exact Solution (if any)
Data.u_ex     = {@(x,y) tanh(-20*(x.^2+y.^2-0.8))};
% Gradient of the Exact Solution
Data.gradu_ex =  {@(x,y) -40*x.*sech(20.*(x.^2+y.^2-4/5)).^2; ...
                  @(x,y) -40*y.*sech(20.*(x.^2+y.^2-4/5)).^2};

%% Adaptivity
Data.Adaptivity = true;                 % Flag of adaptivity                                             
Data.maxDegree  = 6;                    % Maximum polynomial degree
Data.AdaptFunc  = @(tau_r) floor(1 + 2*Data.maxDegree/pi* atan(tau_r));     % Adaptivity function
Data.AdaptIts   = 10;                   % Maximum iterations of adaptivity
\endverbatim

\subsection subUGL3 Run a simulation

With the file \b RunMainLaplacian.m you can run the specific simulation. 

\verbatim
%% RunMainLaplacian.m

%% Initial Simulation Setup
MyPhysicsPath = pwd;
run('../SimulationSetup.m');
%% Input Data - Boundary conditions - Forcing term
DataTestTanh;
%% Mesh Generation
if Data.MeshFromFile
    % Load existing mesh
    Data.meshfile = fullfile(Data.FolderName, Data.meshfileseq);
else
    % Create a new mesh
    Data.meshfile = MakeMeshMonodomain(Data,Data.N,Data.domain,Data.FolderName,Data.meshfileseq,'P','laplacian');
end
%% Main
[Error] = MainLaplacian(Data,Setup);
\endverbatim
