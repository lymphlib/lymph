\section sec_Poisson Poisson's problem 
	
This is a simple tutorial to solve the Poisson problem 

\f[\begin{cases}
        - \nabla \cdot (\mu \nabla u) = f, &  x \in \Omega,\\
        u  =  g, &  x \in \partial \Omega,
  \end{cases}\f]
  
in \f$\Omega =(0,1)^2 \f$ with the following data:

\f{eqnarray*}{
 \bullet \; \mu(x,y) & = & 1, \\
 \bullet \; f(x,y) & = & 8\pi^2\sin(2\pi x)\cos(2\pi y),\\
 \bullet \; g(x,y) & = & \sin(2\pi x)\cos(2\pi y),\\
 \f}
 
whose exact solution is \f$ u(x) = \sin(2\pi x)\cos(2\pi y)\f$.

\section secTutLap1 Setup the input data
   
We set up these data in InputData/DataTestLap.m script. We consider:
    - the number of the element mesh \f$N_{el} = 100 \f$; \n
    - the polynomial approximation degree \f$\ell_\kappa=3\f$ for any \f$\kappa \in \mathcal{T}_h\f$; \n
    - the penalty constant \f$C_\alpha = 10 \f$. \n

We report here for completeness a sample of the script.

\verbatim
%% InputData/DataTestLap.m 
%% Poisson problem in [0,1]^2 with Dirichlet conditions
Data.name = 'TestLapDir';
Data.LabEl = {'L'};     % Element labels
Data.TagEl = { 1 };     % Element tags
Data.TagBc = {[2 3 4 5]};  % Boundary tags
Data.LabBc = {'DDDD'};  % (D)irichlet/(N)eumann/(A)bsorbing
%% Geometrical properties 
Data.domain       = [0 1 0 1]; % domain bounds for a new mesh
Data.N            = 30;        % number of elements for a new mesh
Data.MeshFromFile = false;     % read mesh from file
...
%% Space discretization
Data.degree  = 3;   % Polynomial degree
Data.penalty_coeff = 10; % Penalty coefficient
...
%% Material properties 
Data.mu       = {@(x,y) 1.*x.^0.*y.^0};
%% Forcing Term
Data.source   = {@(x,y) 2*(4*pi^2)*sin(2*pi*x).*cos(2*pi*y)};
%% Boundary Conditions
Data.DirBC    = {@(x,y) sin(2*pi*x).*cos(2*pi*y)};
% Exact Solution (if any)
Data.u_ex     =  {@(x,y) 5*sin(2*pi*x).*cos(2*pi*y)};
% Gradient of the Exact Solution
Data.gradu_ex =  {@(x,y)  10*pi*cos(2*pi*x).*cos(2*pi*y); ...
                  @(x,y) -10*pi*sin(2*pi*x).*sin(2*pi*y)};
%% Adaptivity
Data.Adaptivity = false;                 % Flag of adaptivity
...               
\endverbatim

\verbatim
%% RunMainLaplacian.m
%% Initial Simulation Setup
MyPhysicsPath = pwd;
run('../SimulationSetup.m');
%% Input Data - Boundary conditions - Forcing term
%DataTestTanh;
DataTestLap;
%% Mesh Generation
if Data.MeshFromFile
    % Load existing mesh
    Data.meshfile = fullfile(Data.FolderName, Data.meshfileseq);
else
    % Create a new mesh
    Data.meshfile = MakeMeshMonodomain(Data,Data.N,Data.domain,Data.FolderName,Data.meshfileseq,'P','laplacian');
end
%% Main
[Error] = MainLaplacian(Data,Setup)
\endverbatim 
 
The script contains different calls, among which one is devoted to the mesh construction, i.e., \b MakeMeshMonodomain.m
that is contained in \b Core/MeshGeneration.
\verbatim
Data.meshfile = MakeMeshMonodomain(Data,Data.N,Data.domain,Data.FolderName,Data.meshfileseq,'P','laplacian');
\endverbatim 
The function \b MakeMeshMonodomain.m calls the PolyMesher program  \cite talischi2012polymesher to generate the polygonal mesh for the considered domain.
The mesh structure is saved in \b Data.meshfile.

\section secpAdaptive Adaptive framework

It is possible to exploit the \f$\ell\f$-adaptive PolyDG framework in order to employ different polynomial approximation degrees on different elements of the computational mesh.
The use of a non-uniform polynomial distribution allows the numerical method to increase the approximation order only in regions where is needed, while keeping a lower degree elsewhere.
This leads to a more efficient use of the available degrees of freedom compared to uniform high-order discretizations.

The \f$\ell\f$-adaptive strategy is driven by a local a-posteriori error indicator from \cite cangiani_posteriori_2023, computed
element-wise from the discrete solution. We introduce an element-wise a-posteriori indicator used to quantify the local quality of the numerical approximation.
Given the discrete solution \f$u_h\f$, the local indicator \f$\tau_K\f$ is defined as the combination of the following
contributions:  
\f[
\tau_K  =
\sum_{K \in \mathcal{T}_h}
\left( R_K^2 + O_K^2 \right),
\f]
The local error estimator \f$R_K\f$ and the data oscillation term \f$O_K\f$ are defined
element-wise and decompose as
\f[
R_K^2 = R_{K,E}^2 + R_{K,N}^2 + R_{K,J}^2 + R_{K,T}^2,
\qquad
O_K^2 = O_{K,E}^2 + O_{K,N}^2 + O_{K,J}^2 + O_{K,T}^2 \;.
\f]
Each term is detailed below.
- \b Residual \b component: The element residual contribution measures the violation of the strong form of the
    Poisson equation inside each element:
    \f[
    R_{K,E}
    =
    \left\|
    h_K \bigl( \Pi f + \nabla \cdot (\mu \nabla u_h) \bigr)
    \right\|_{L^2(K)} .
    \f]
    Here, \f$h_K\f$ denotes the diameter of element \f$K\f$, and \f$\Pi f\f$ is the local polynomial projection of the forcing term.

- \b Jump \b component: The jump of the discrete solution across interior faces is measured by:
    \f[
    \tau_{K,j}(u_h^{(k)})
    =
    \left\|
    \sqrt{\eta}
    [\![{u_h^{(k)}}]\!]
    \right\|_{L^2(\partial K \cap \mathcal{F}_h^I)} ,
    \f]
    where \f$\eta\f$ is the DG penalty parameter.
    This term is directly related to the stability mechanism of the discontinuous
    Galerkin formulation.

- \b Normal \b flux \b jump \b component: The normal flux jump term accounts for the lack of continuity of the numerical
    flux across element interfaces:
    \f[
    \tau_{K,n}(u_h^{(k)})
    =
    \left(
    \|
    \sqrt{h_K}
    [\![ {\boldsymbol{\Sigma} \nabla_h u_h^{(k)} \cdot \boldsymbol{n}_K} ]\!]
    \|_{L^2(\partial K \cap \mathcal{F}_h^I)}^2
    +
    \|
    \sqrt{h_K}
    (\boldsymbol{\Sigma} \nabla_h u_h^{(k)} \cdot \boldsymbol{n}_K)
    \|_{L^2(\partial K \cap \mathcal{F}_h^N)}^2
    \right)^{1/2}.
    \f]
    Here, \f$\mathcal{F}_h^I\f$ and \f$\mathcal{F}_h^N\f$ denote the sets of interior and
    Neumann boundary faces, respectively, and \f$\boldsymbol{n}_K\f$ is the outward unit
    normal to \f$\partial K\f$.

- \b Tangential \b flux \b jump \b component: tangential component of the flux jump is included to enhance the
sensitivity of the indicator to anisotropic solution features:
\f[
\tau_{K,t}(u_h^{(k)})
=
\left\|
\sqrt{h_K}
{\boldsymbol{\Sigma} \nabla_h u_h^{(k)} \cdot \boldsymbol{t}_K}
\right\|_{L^2(\partial K \cap \mathcal{F}_h^I)} ,
\f]
where \f$\boldsymbol{t}_K\f$ denotes a unit tangential vector along the faces of
element \f$K\f$.

The data oscillation terms account for the mismatch between the physical data and
their polynomial projections:
\f[
\begin{aligned}
O_{K,E} &= \| h_K (f - \Pi f) \|_{L^2(K)}, \quad\quad\;\quad O_{K,N} = \| \sqrt{h_K} (g_N - \bar{g}_N) \|_{L^2(\partial K \cap \mathcal{F}_h^N)}, \\
O_{K,J} &= \left\| \sqrt{\eta} (g_D - \bar{g}_D) \right\|_{L^2(\partial K \cap \mathcal{F}_h^D)}, \quad O_{K,T} = \| \sqrt{h_K} \nabla_T (g_D - \bar{g}_D) \|_{L^2(\partial K \cap \mathcal{F}_h^D)} .
\end{aligned}
\f]
In the \f$\ell\f$-adaptive framework, the element-wise indicator is mapped into a target polynomial degree through an adaptivity function.
This function translates the magnitude of the local estimator into an appropriate approximation order, allowing higher-degree polynomials to be employed only on 
elements where the indicator is high. For the Poisson problem, a specific adaptivity function is adopted.
However, this choice is not restrictive, and alternative mappings between the local indicator and the polynomial degree can be defined and tuned using problem-dependent data.

\subsection pAdaptiveSetup Setup Data 
The setup of the \f$\ell\f$-adaptive simulations is handled through dedicated input data files. The script we exploit is DataTestTanh.m.
\verbatim
%% InputData/DataTestTanh.m 
...
%% Adaptivity
Data.Adaptivity = true;                 % Flag of adaptivity                                             
Data.maxDegree  = 6;                    % Maximum polynomial degree
Data.AdaptFunc  = @(tau_r) floor(1 + 2*Data.maxDegree/pi* atan(tau_r));     % Adaptivity function
Data.AdaptIts   = 10;                   % Maximum iterations of adaptivity
...
\endverbatim

The flag \b Data.Adaptivity activates or deactivates the \f$\ell\f$-adaptive procedure. When
set to \c true, the polynomial degree is allowed to vary locally from element to
element according to a either the a-posteriori error indicator or the predefined distribution of polynomial degree as input.
The parameter \b Data.maxDegree specifies the maximum admissible polynomial degree
\f$\ell_\text{max}\f$. During the adaptive process, no element is assigned a polynomial
degree larger than this value, independently of the magnitude of the local indicator.
The mapping between the local indicator and the target polynomial degree is defined
by \b Data.AdaptFunc. Finally, the parameter \b Data.AdaptIts sets the maximum number of adaptivity
iterations allowed in the simulation.
