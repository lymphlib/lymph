\section sec_FHN Monodomain equation coupled with the FitzHugh--Nagumo ionic model

This tutorial presents the solution of the monodomain equation coupled with the FitzHugh--Nagumo ionic model using the PolyDG method implemented in the <tt>Physics/FitzHughNagumo</tt> folder.
The mathematical model reads
\f[
\chi_m C_m \frac{\partial u}{\partial t}
-
\nabla_m \cdot (\boldsymbol{\Sigma} \nabla u)
+
\chi I_{\mathrm{ion}}(u,w)
=
I_{\mathrm{app}},
\qquad {\rm in} \;\Omega \times (0,T],
\f]
coupled with the ionic variable equation
\f[
\frac{\partial w}{\partial t}
=
g(u,w),
\qquad {\rm in} \;\Omega \times (0,T].
\f]

Here, \f$u\f$ denotes the transmembrane potential, \f$w\f$ is the recovery variable of the FitzHugh-Nagumo model, \f$\boldsymbol{\Sigma}\f$ is the conductivity tensor, \f$\chi_m\f$ is the membrane surface-to-volume ratio, and \f$C_m\f$ is the membrane capacitance.

The ionic model is described by the FitzHugh--Nagumo reaction terms
\f[
I_{\mathrm{ion}}(u,w)
=
k\,u(u-a)(u-1) + w,
\f]
and
\f[
g(u,w)
=
\varepsilon(\beta u - \gamma w).
\f]

Here, \f$a\f$, \f$k\f$, \f$\varepsilon\f$, \f$\beta\f$, and \f$\gamma\f$ are model parameters regulating the excitation and recovery dynamics of the membrane potential. As a benchmark application of the presented PolyDG method, we consider a wave propagation problem on a two-dimensional domain.
The initial condition and the applied stimulus generate an excitation wave that propagates through the computational domain according to the monodomain dynamics coupled with the FitzHugh--Nagumo ionic response.

\section secTutFHN1 Setup the input data

We set up these data in the script \b InputData/DataTestCaseFHN.m.
In particular, we consider:
    - the computational domain \f$\Omega = (-1,1)^2\f$ with Dirichlet boundary conditions; \n
    - the FitzHugh--Nagumo ionic parameters
    \f$a = 0.5\f$,
    \f$\varepsilon = 1\f$,
    \f$\beta = 1\f$,
    \f$\gamma = 2\f$,
    \f$k = 1\f$; \n
    - the final simulation time \f$T = 2\f$; \n
    - the time integration step \f$\Delta t = 10^{-2}\f$; \n
    - the initial time \f$t_0 = 0\f$ and the time discretization parameter \f$\theta = 1\f$; \n
    - the number of mesh elements \f$N_{el} = 180\f$; \n
    - the polynomial approximation degree \f$\ell_\text{max} = 3\f$;
    - the penalty coefficient \f$C_\alpha = 10\f$; \n
    - Picard iterations for the nonlinear solver, with tolerance \f$10^{-10}\f$, maximum number of iterations equal to \f$1000\f$, and relaxation parameter equal to \f$1\f$; \n

We report here for completeness a sample of the script.

\section secAdaptiveFHN l-adaptive framework
The \f$\ell\f$-adaptive strategy is driven by a local a-posteriori residual indicator, computed element-wise from the discrete solution, following \cite cangiani_posteriori_2023.
A detailed description of the estimator components is provided in \ref secAdaptiveHeat. Here, the indicator is extended to the time-dependent heat equation by accounting for both the current and previous time levels through the \f$\theta\f$-time discretization.

\subsection pAdaptiveSetupFHN Setup Data 
We set up these data in the script \b InputData/DataSpiral.m.
In particular, we consider:
    - the computational domain \f$\Omega = (0,2.5)^2\f$ with homogeneous Neumann boundary conditions; \n
    - the FitzHugh--Nagumo ionic parameters
    \f$a = 0.1\f$,
    \f$\varepsilon = 0.01\f$,
    \f$\beta = 1\f$,
    \f$\gamma = 0.5\f$,
    \f$k = 10\f$; \n
    - the final simulation time \f$T = 2000\f$; \n
    - the time integration step \f$\Delta t = 5 \cdot 10^{-1}\f$; \n
    - the number of mesh elements \f$N_{el} = 2500\f$; \n
    - the penalty coefficient \f$C_\alpha = 10\f$; \n

\verbatim
%% InputData/DataSpiral.m 
%% Simulation for FHN equation
Data.name = 'FHN_Adaptive';
Data.LabEl = {'N:q'};     % Element labels
Data.TagEl = { 1 };     % Element tags
Data.TagBc = {[2 3 4 5]};  % Boundary tags
Data.LabBc = {'NNNN'};  % (D)irichlet/(N)eumann/(A)bsorbing
Data.TagApplyBCs = 0;      % Skip the assembling of BCs if not necessary
%% Geometrical properties
Data.domain          = [0 2.5 0 2.5];
Data.N               = [50 50]; 
...
% Exact Solution
Data.u_ex = {@(x,y,t) double( (x <= 1.25) & (y <= 1.25) )};
eps = 0.09;
Data.w_ex = {@(x,y,t) 0.1 * 0.5 * (1 + tanh((y - 1.25)/eps))};
...
%% Nonlinear solver settings
Data.NonLinearSolver = 'Picard iterations';     % Nonlinearity treatment: Semi-implicit or Picard iterations
Data.NLS_tolerance  = 1e-10;
Data.NLS_max_it     = 1000;
Data.NLS_relax      = 1;
...
%% Adaptivity
Data.Adaptivity         = true;                % Flag of adaptivity                                             
Data.maxDegree          = 4;                    % Maximum polynomial degree
Data.AdaptFunc          = @(tau_r) floor(1 + (Data.maxDegree-1)*tanh(5*tau_r));     % Adaptivity function
Data.AdaptIts           = 2;                   % Maximum iterations of adaptivity
Data.AdaptivityStep     = 10; 
\endverbatim

The parameter \b Data.AdaptivityStep defines the frequency of the adaptive update during the time integration procedure.

\subsection pAdaptiveSimulationFHN Simulation Run

We run the specific simulation using the script \b RunMainFHN.m, which calls the main algorithm, contained in \b MainFunctions/MainFHN.m, and the Dataset \b DataSpiral.m 
As output of the run, we first obtain a set of plots associated with the polygonal mesh \f$\mathcal{T}_h\f$ and the corresponding numerical solution.
For this particular simulation we exploited a polytopal mesh of 2500 elements and \f$\ell_\text{max}=4\f$.

%% Initial Simulation Setup

\verbatim
[Data.meshfile] = MakeMeshMonodomain(Data,Data.N,Data.domain,Data.FolderName,Data.meshfileseq,'C','laplacian');
\endverbatim
The function \b MakeMeshMonodomain.m can generate different types of polygonal meshes depending on the selected option.
In this case, the flag \c 'C' is used to construct a Cartesian polygonal mesh for the considered domain.
